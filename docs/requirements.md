## 1. 项目概述

### 1.1 项目背景

GIO（光里光外）成立于 2010 年，是一家专注于智能照明全案设计的知名公司。随着业务发展，需要建立一套数字化展示与管理系统，用于：

- 展示公司作品案例，提升品牌形象
- 方便客户了解公司专业领域和过往项目
- 内部管理项目资料和案例图片

### 1.2 项目定位

- **C 端官网**：面向潜在客户和合作伙伴的展示网站
- **管理后台**：面向内部运营人员的内容管理系统

### 1.3 系统架构

```
│                         用户访问                             │

        ▼                                           ▼
┌───────────────────┐                    ┌───────────────────┐
│   C 端官网 (8081)  │                    │  管理后台 (8082)   │
│   gio-portal      │                    │   gio-admin       │

```

## 2. 数据库设计

| sort_order | INT | 排序权重 |

| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**预置分类**：
- 住宅空间 (residential)
- 餐饮空间 (restaurant)
- 娱乐空间 (entertainment)
- 办公空间 (office)
- 酒店空间 (hotel)
- 婚礼空间 (wedding)
- 会所空间 (club)
- 医疗空间 (medical)
- 展厅空间 (exhibition)

| 字段 | 类型 | 说明 |

## 3. C 端用户官网

### 3.1 访问地址
- **官网网址访问地址**：http://localhost:8888
- **官网后端服务域名地址**：http://localhost:8081

### 3.2 页面结构

```
首页
├── 首页 (Home)
├── 案例作品 (Projects)
│   ├── 项目列表（支持分类筛选）
│   └── 项目详情
├── 关于我们 (About)
└── 联系我们 (Contact)
```

### 3.3 功能需求

#### 3.3.1 首页 (Home)

**页面组成**：

| 区块 | 内容 | 说明 |
|------|------|------|
| Hero Banner | 全屏大图 + 品牌标语 | 展示"光里光外 GIO - 智能照明全案设计公司" |
| 专业领域 | 6个分类卡片 | 点击跳转至对应分类的项目列表 |
| 精选作品 | 6个推荐项目卡片 | 点击跳转至项目详情 |
| 关于我们 | 公司简介 | 包含简短介绍和"了解更多"链接 |

**交互说明**：
- 页面加载时自动获取前6个分类
- 精选作品默认获取前6个项目
- 分类卡片支持hover动画效果
- 项目卡片支持hover放大效果

#### 3.3.2 案例作品页 (Projects)

**功能列表**：
- 显示所有已发布项目
- 支持分类筛选（全部 + 各分类）
- 分页加载

**页面元素**：
- 页面顶部标题区
- 分类筛选按钮（横向滚动）
- 项目卡片网格（响应式：1/2/3列）
- 空状态提示

**项目卡片信息**：
- 项目封面图片
- 分类名称
- 项目名称
- 位置 + 年份

#### 3.3.3 项目详情页 (ProjectDetail)

**页面组成**：

| 区块 | 内容 |
|------|------|
| 顶部信息区 | 项目名称、分类、年份、位置、浏览量 |
| 主图展示 | 大图展示，支持点击切换 |
| 缩略图栏 | 图片缩略图列表 |
| 项目描述 | 详细的项目说明文字 |
| 联系我们 | CTA 引导用户联系 |

**交互说明**：
- 支持图片切换（点击缩略图）
- 自动记录浏览量
- 返回按钮可返回项目列表

#### 3.3.4 关于我们页 (About)

**内容**：
- 公司成立时间（2010年）
- 公司定位和理念
- 服务特色介绍

#### 3.3.5 联系我们页 (Contact)

**内容**：
- 联系方式展示
- 公司地址信息
- 在线留言表单（预留）

## 4. 管理后台

### 4.1 访问地址
- **后台网址访问地址**：http://localhost:8888/admin
- **后台后端服务域名地址**：http://localhost:8082

### 4.2 页面结构

```
管理后台
├── 登录页 (Login)
└── 内容管理
    ├── 仪表盘 (Dashboard)
    ├── 项目管理 (Projects)
    ├── 分类管理 (Categories)
    └── 图片管理 (Images)
```

### 4.3 功能需求

#### 4.3.1 登录功能

**功能说明**：
- 管理员账号密码登录
- JWT Token 认证
- Token 有效期 24 小时

**登录凭证**：
- 默认账号：admin
- 默认密码：admin123

**交互流程**：
1. 输入用户名和密码
2. 点击登录按钮
3. 验证成功后返回 JWT Token
4. 前端存储 Token 到 localStorage
5. 后续请求携带 Token

#### 4.3.2 仪表盘 (Dashboard)

**功能说明**：
- 展示系统核心数据统计

**数据展示**：
- 项目总数
- 已发布项目数
- 分类总数
- 图片总数

#### 4.3.3 项目管理 (Projects)

**功能列表**：

| 功能 | 说明 |
|------|------|
| 项目列表 | 显示所有项目（含草稿），支持分页 |
| 创建项目 | 新建项目基本信息 |
| 编辑项目 | 修改项目信息 |
| 删除项目 | 删除项目（级联删除图片） |
| 上传图片 | 为项目上传多张图片 |
| 设置封面 | 设置项目封面图片 |
| 发布/下架 | 切换项目状态 |

**列表字段**：
- ID
- 项目名称
- 所属分类
- 位置
- 年份
- 状态（已发布/草稿）
- 操作

**项目表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目名称 | 文本 | 是 | |
| 所属分类 | 下拉选择 | 是 | |
| 位置 | 文本 | 否 | |
| 年份 | 文本 | 否 | 如：2024 |
| 项目描述 | 多行文本 | 否 | |
| 排序权重 | 数字 | 否 | 越小越靠前 |

#### 4.3.4 分类管理 (Categories)

**功能列表**：

| 功能 | 说明 |
|------|------|
| 分类列表 | 显示所有分类 |
| 创建分类 | 新建分类 |
| 编辑分类 | 修改分类信息 |
| 删除分类 | 删除分类 |

**分类表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 分类名称 | 文本 | 是 | 中文名称 |
| 英文名称 | 文本 | 否 | English Name |
| 分类编码 | 文本 | 是 | 小写英文，如 residential |
| 图标 | 文本 | 否 | 图标路径 |
| 排序权重 | 数字 | 否 | 越小越靠前 |
| 状态 | 开关 | 是 | 启用/禁用 |

#### 4.3.5 图片管理 (Images)

**功能说明**：
- 上传图片到项目
- 图片支持 JPG、PNG 格式
- 自动生成缩略图（可选）

**上传说明**：
- 文件大小限制：10MB
- 支持多文件上传
- 上传目录：./uploads/projects/

## 5. 安全性设计

### 5.1 认证与授权

- JWT Token 认证
- Token 有效期：24 小时
- 密码加密存储（BCrypt）

### 5.2 接口安全

- 管理后台接口需要登录认证
- C 端接口公开访问
- 敏感操作日志记录

### 5.3 文件安全

- 上传目录与应用分离
- 文件类型校验
- 文件大小限制

## 6. 非功能性需求

### 6.1 性能要求

- 页面加载时间 < 3 秒
- API 响应时间 < 500ms
- 支持并发访问

### 6.2 兼容性

- Chrome、Firefox、Safari、Edge 最新版本
- 移动端响应式适配

### 6.3 可用性

- 友好的错误提示
- 加载状态指示
- 操作反馈明确
