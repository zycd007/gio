# 项目管理页面卡片展示重构设计

**日期**: 2026-04-06
**状态**: 已确认

## 背景

当前后台管理项目管理页面（`gio-web/src/admin/Projects.tsx`）使用表格形式展示项目列表。用户希望改为卡片展示形式，提升视觉体验和操作效率。

## 设计目标

- 将项目列表从表格改为卡片展示
- 重新设计整体 UX 交互，做到好看实用
- 保持现有功能完整性（筛选、搜索、分页、操作等）

## 设计方案

### 1. 卡片风格：状态指示型

卡片顶部设置状态栏，一眼识别项目状态：

- **已发布状态**：绿色边框 (#10b981)，绿色状态栏，显示"✓ 已发布"
- **草稿状态**：橙色边框 (#f59e0b)，橙色状态栏，显示"📝 草稿"
- **精品项目**：状态栏右侧显示金色标签"⭐ 精品"

### 2. 卡片布局：响应式自适应

```
grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
gap: 20px;
```

- 卡片最小宽度 280px，自动填充行
- 不同屏幕宽度自适应列数
- 大屏幕约 4 列，中等屏幕约 3 列，小屏幕 2 列或 1 列

### 3. 卡片交互：Hover 显示操作

鼠标悬停封面区域时显示快捷操作按钮：

- **查看按钮**：进入项目详情页
- **编辑按钮**：打开编辑弹窗
- **更多按钮**：下拉菜单包含：
  - 图片管理
  - AI 文案生成
  - 发布/下架切换
  - 设为/取消精品
  - 删除

### 4. 顶部工具栏：两行布局

**第一行**：
- 左侧：页面标题"项目管理" + 统计数字"共 X 个项目"
- 右侧：新建项目按钮（渐变绿色，带阴影）

**第二行**：
- 搜索框（最大宽度 320px）
- 分类筛选下拉
- 状态筛选下拉（已发布/草稿）
- 精品筛选下拉

### 5. 卡片内容结构

```
┌─────────────────────────────┐
│ ✓ 已发布      ⭐ 精品        │ ← 状态栏（绿色/橙色）
├─────────────────────────────┤
│                             │
│      [项目封面图片]          │ ← 160px 高度
│   (hover 显示操作按钮)       │
│                             │
├─────────────────────────────┤
│ 成都私宅照明设计             │ ← 项目名称
│ 私宅 · 📍 成都 · 2024       │ ← 分类 + 位置 + 年份
└─────────────────────────────┘
```

### 6. 分页设计

保持现有分页功能，调整样式：
- 底部固定区域
- 左侧显示"显示 1-12 / 共 X 项"
- 右侧显示分页按钮（上一页、页码、下一页）

## 关键文件

- **主要修改文件**: `gio-web/src/admin/Projects.tsx`
- **保持不变**: `gio-web/src/admin/ProjectDetailPage.tsx`（项目详情页）
- **样式参考**: `gio-web/src/pages/Projects.tsx`（前端展示页面的卡片风格）

## 技术实现要点

1. **移除表格结构**：删除 `<table>` 相关代码，改为 `grid` 布局
2. **新增卡片组件**：创建 `ProjectCard` 子组件处理单个卡片渲染
3. **Hover 交互**：使用 CSS `group-hover` 或 React 状态管理显示操作层
4. **状态边框**：根据 `project.status` 和 `project.isFeatured` 动态设置边框颜色
5. **工具栏重构**：调整为两行布局结构
6. **保持 API 调用**：所有现有 API 接口保持不变

## 验收标准

1. 项目以卡片形式展示，视觉效果美观
2. 状态区分清晰（绿色=已发布，橙色=草稿）
3. Hover 操作流畅，按钮响应正常
4. 筛选、搜索、分页功能正常工作
5. 所有原有操作功能可用（查看、编辑、图片管理、AI 文案、状态切换、删除）
6. 响应式布局在不同屏幕下表现良好

## 风险与注意事项

- 卡片布局占用更多垂直空间，需确保分页合理
- Hover 操作在移动端需考虑触摸替代方案（点击显示操作）
- 确保图片加载失败时有占位图处理
