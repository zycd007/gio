# ux-review

对 GIO 项目前端进行用户体验走查，输出问题清单和优化建议。

## 使用方式

```
/ux-review [页面路径]
```

**参数说明：**
- 无参数 - 对全站进行走查
- `home` - 首页走查
- `projects` - 项目列表页走查
- `project-detail` - 项目详情页走查
- `about` - 关于我们页走查
- `contact` - 联系我们页走查
- `admin` - 管理后台整体走查

## 走查范围

### 1. 视觉一致性

| 检查项 | 说明 |
|--------|------|
| 颜色规范 | 主色、辅色、状态色使用是否一致 |
| 字体规范 | 字号、字重、行高是否统一 |
| 间距规范 | margin/padding 是否遵循栅格系统 |
| 组件样式 | 按钮、卡片、表单样式是否统一 |
| 图标使用 | 图标风格、大小是否一致 |

### 2. 交互体验

| 检查项 | 说明 |
|--------|------|
| 操作反馈 | 点击、提交是否有明确的视觉/文案反馈 |
| 加载状态 | 数据加载中是否有 loading 指示 |
| 错误处理 | 表单验证、网络错误是否有友好提示 |
| 空状态 | 数据为空时是否有占位提示 |
| 导航清晰 | 用户能否清楚知道当前位置 |
| 操作确认 | 危险操作（删除等）是否有二次确认 |

### 3. 可访问性

| 检查项 | 说明 |
|--------|------|
| 键盘导航 | 是否可以通过键盘完成主要操作 |
| 对比度 | 文字与背景对比度是否符合 WCAG 标准 |
| ARIA 标签 | 交互元素是否有正确的 aria 属性 |
| 焦点管理 | 焦点状态是否可见，焦点顺序是否合理 |
| 屏幕阅读器 | 重要内容是否能被正确读取 |

### 4. 响应式设计

| 检查项 | 说明 |
|--------|------|
| 移动端适配 | 375px 宽度下布局是否正常 |
| 平板适配 | 768px 宽度下布局是否正常 |
| 桌面适配 | 1024px+ 宽度下布局是否正常 |
| 图片响应式 | 图片是否根据屏幕尺寸适配 |
| 触摸友好 | 按钮和链接的触摸区域是否足够大 |

### 5. 性能体验

| 检查项 | 说明 |
|--------|------|
| 首屏加载 | 首屏内容是否快速渲染 |
| 图片优化 | 图片是否懒加载、是否使用合适格式 |
| 动画流畅 | 动画是否使用 GPU 加速 |
| 代码分割 | 是否按路由懒加载组件 |

### 6. 内容体验

| 检查项 | 说明 |
|--------|------|
| 文案清晰 | 文案是否简洁易懂 |
| 信息层级 | 信息架构是否清晰 |
| 行动召唤 | CTA 按钮是否突出明确 |
| 帮助提示 | 复杂操作是否有引导说明 |

## 页面清单

### C 端页面

| 页面 | 路径 | 主要检查点 |
|------|------|------------|
| 首页 | `/` | 视觉冲击力、信息层级、行动召唤 |
| 项目列表 | `/projects` | 筛选交互、卡片布局、加载状态 |
| 项目详情 | `/projects/:id` | 图片展示、信息组织、返回导航 |
| 关于我们 | `/about` | 内容阅读体验、团队展示 |
| 联系我们 | `/contact` | 表单体验、验证反馈、提交状态 |

### 管理后台

| 页面 | 路径 | 主要检查点 |
|------|------|------------|
| 登录 | `/admin/login` | 表单体验、错误提示、记住登录 |
| 仪表盘 | `/admin` | 数据展示、快捷操作 |
| 项目管理 | `/admin/projects` | 列表操作、批量操作、状态切换 |
| 分类管理 | `/admin/categories` | 拖拽排序、增删改操作 |
| 留言管理 | `/admin/messages` | 列表展示、状态标记、详情查看 |
| 文案管理 | `/admin/copywriting` | 编辑体验、AI 辅助功能 |

## 输出格式

走查完成后，按以下格式输出：

### 问题清单

| 序号 | 页面 | 问题类型 | 问题描述 | 严重程度 | 建议方案 |
|------|------|----------|----------|----------|----------|
| 1 | 首页 | 交互 | 首屏加载无 loading 指示 | 高 | 添加骨架屏 |
| 2 | 项目详情 | 视觉 | 图片比例不一致 | 中 | 统一使用 4:3 比例 |

### 严重程度说明

- **高**: 影响核心功能使用，需立即修复
- **中**: 影响用户体验，建议近期修复
- **低**: 细节优化，可择机处理

### 优化建议

按优先级列出具体的优化建议，包括：
- 问题描述
- 影响范围
- 推荐方案
- 参考代码位置

## 执行步骤

1. **确定走查范围** - 根据参数确定检查的页面
2. **阅读代码** - 分析相关页面组件代码
3. **模拟用户操作** - 按检查清单逐一检查
4. **记录问题** - 发现问题立即记录
5. **输出报告** - 汇总问题并给出建议

## 参考文件

| 文件 | 说明 |
|------|------|
| `gio-web/src/App.tsx` | 路由配置 |
| `gio-web/src/pages/*.tsx` | C 端页面组件 |
| `gio-web/src/admin/*.tsx` | 管理后台页面 |
| `gio-web/src/components/*.tsx` | 公共组件 |
| `gio-web/tailwind.config.js` | 样式规范配置 |

## 注意事项

- 走查基于代码分析，建议结合实际运行效果验证
- 关注真实用户场景，而非理论情况
- 优先考虑核心用户流程
- 建议多人参与走查，获取不同视角
